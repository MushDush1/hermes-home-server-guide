# 04 - Permissions and Safety

高自治的重点不是“放开所有权限”，而是让 Hermes 可以连续推进任务，同时在关键边界自动停住。

## 权限等级

| Level | 名称 | 允许行为 | 例子 |
| --- | --- | --- | --- |
| L0 | observe | 只读、总结、分类 | 读网页、读本地允许目录、总结日志 |
| L1 | draft | 生成草稿，不外发 | 起草微信回复、生成计划、生成代码 patch 草稿 |
| L2 | low_risk_act | 白名单低风险动作 | 写入 scratch 目录、生成报告、跑只读检查 |
| L3 | guarded_act | 需要审批的动作 | 改仓库文件、运行测试、调用 Codex 写代码 |
| L4 | sensitive_act | 强审批动作 | 发外部消息、删文件、账号操作、花钱 |
| L5 | financial_act | 默认禁止或强审批 | 真实下单、跟单、转账、改券商设置 |
| L6 | private_learning | 强审批或禁用 | 使用登录态读取私密内容、批量抓平台内容 |

默认策略：

- L0-L1 可自动执行。
- L2 只允许白名单。
- L3-L4 必须审批。
- L5 第一阶段禁止；后续即使开放，也必须独立二次确认。
- L6 默认只允许你明确授权的单次任务，不允许后台定时任务静默执行。

## 决策授权等级

权限回答“能不能做”，决策授权回答“能不能自己拍板”。两者要同时通过。

| Decision Level | 名称 | Hermes 是否能决定 | 例子 |
| --- | --- | --- | --- |
| D0 | explain_only | 不能决定，只解释 | 投资教育、法律/医疗概念解释 |
| D1 | recommend_only | 推荐但等待你选 | 选择理财方向、是否接管微信账号 |
| D2 | autonomous_low | 可自动决定 | 选择模型、整理学习队列、报告格式 |
| D3 | autonomous_reversible | 可先做后报 | 只读任务失败后重试一次、生成模拟组合笔记 |
| D4 | approval_required | 必须审批 | 改文件、发消息、访问登录态、读完整财务资料 |
| D5 | forbidden | 禁止决策 | 真实交易、转账、冒充你、绕过平台限制 |

决策输入必须包含：

- 风险等级。
- 可逆性。
- 置信度。
- 是否对外代表你。
- 是否涉及钱、账号、隐私、真实交易。
- 失败成本。

Hermes 不能用“我建议你...”逃避所有选择。对于 D2-D3，它应该直接说：

```text
我决定采用 X，因为 Y。风险是 Z。如果结果不好，回滚方式是 R。
```

对于 D4，它应该说：

```text
我推荐 X，但这需要你的审批，因为它会影响 Y。
```

## 高风险动作清单

以下动作必须进入审批队列：

- 发 QQ/微信/邮件/短信。
- 删除、移动、覆盖非 scratch 目录文件。
- 修改 git 仓库。
- 运行未知命令。
- 安装软件包。
- 访问浏览器登录态执行操作。
- 使用支付、下单、账号设置、隐私资料。
- 接管或模拟你的个人身份发言。
- 读取完整持仓、收入、负债等私密金融资料。
- 生成个性化买卖建议。
- 创建真实交易订单、自动跟单、调仓或转账。
- 使用“保证收益”“稳赚”“无风险高回报”等表述。
- 批量下载视频/音频作为训练资料。
- 使用登录态静默读取收藏夹、私信、关注流或付费内容。
- 绕过平台访问限制、验证码、反爬或版权保护。

## CLI 执行隔离

所有外部 agent/CLI 必须有：

- 固定 `cwd`。
- 超时。
- stdout/stderr 捕获。
- 退出码捕获。
- 环境变量 allowlist。
- 文件写入边界。
- 任务级 trace id。

建议目录：

```text
HERMES_DATA_DIR/
  workspaces/
    task_<id>/
      input/
      output/
      logs/
      scratch/
```

默认不允许 CLI 直接在 Hermes 源码目录或你的个人资料目录运行。

## 微信策略

第一阶段使用独立 Hermes 微信账号，不接管主号。

原因：

- 降低误发风险。
- 降低账号封禁影响面。
- 明确“这是 Hermes 的话”，避免身份混淆。
- 方便做权限和联系人白名单。

独立账号建议：

- 只加白名单联系人。
- 群聊默认只读。
- 自动发送默认关闭。
- 所有外发消息先生成草稿和审批卡片。

## 审批消息格式

审批卡片应该短，但信息完整：

```text
Hermes 请求审批

任务：帮我回复 X
动作：发送微信消息
对象：联系人别名
内容预览：...
风险：外部发送，不可完全回滚

回复：
同意 approval_123
拒绝 approval_123
修改 approval_123 <新要求>
```

## 失败处理

外部执行器失败时：

- 标记 Task 为 failed 或 needs_review。
- 保留 stdout/stderr。
- 总结失败原因，但不要编造。
- 如果可重试，给出重试方案。
- 如果涉及高风险动作，不自动重试。

## 记忆安全

长期记忆禁止保存：

- API key、cookie、密码。
- 未确认的猜测。
- 一次性聊天情绪。
- 他人隐私。
- 未经确认的联系人敏感信息。

长期记忆允许保存：

- 你明确确认的偏好。
- 项目路径和架构事实。
- 常用任务流程。
- 经常失败的坑和修复方式。
- 模型/执行器健康状态。

## 金融安全边界

Hermes 的金融能力默认定位为“理财老师”和“研究助理”，不是持牌投资顾问，也不是自动交易机器人。

允许自动做：

- 解释基本概念：资产配置、分散化、风险承受能力、复利、基金费用。
- 整理你的预算和现金流。
- 对公开数据做摘要。
- 帮你建立观察清单。
- 维护模拟组合。
- 用历史数据解释风险，但不能承诺未来结果。

必须审批或禁止：

- 个性化买入/卖出/持有建议。
- 对单只股票、基金、币种给确定性结论。
- 真实交易、转账、借贷、杠杆、期权、合约。
- 让外部 agent 自动操作券商或交易所账号。

输出金融内容时必须包含：

```text
用途：教育 / 研究 / 模拟，不是正式财务建议
数据来源：
数据时间：
关键假设：
主要风险：
下一步建议：需要人工判断
```

## 向外学习安全边界

Hermes 可以学习公开资料、你提供的链接和可合法访问的字幕/摘要，但不能把“能访问”当成“能随便抓”。

允许自动做：

- 读取公开官方文档。
- 读取公开网页并生成摘要。
- 读取你手动提供的链接列表。
- 获取公开视频的标题、简介、发布时间、作者等元数据。
- 在合法可访问时提取字幕做摘要。
- 生成长期记忆候选，等待审核。

必须审批：

- 使用你的浏览器登录态。
- 读取收藏夹、稍后再看、关注列表、私信或付费内容。
- 批量处理同一平台大量内容。
- 保存原始字幕全文、评论全文或大段版权文本。

默认禁止：

- 下载视频/音频作为知识库原料。
- 绕过平台限制或验证码。
- 把版权内容全文写入长期记忆。
- 用定时任务静默操作私人账号。
